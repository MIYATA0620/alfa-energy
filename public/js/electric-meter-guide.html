⚡ Electric Meter Module — Integration Guide

Alfamart Energy Checklist · Patch for energy NaN bug + new electric meter feature

🐛 What's Fixed

✓ Fixed Bug Energy consumption showing NaN

Root cause: dashboard.js defines APPLIANCES with a.hrs property. mobile.js (loaded after) redefines APPLIANCES with a.hours. Since JS uses the last declaration, calcKwhDay(a) in dashboard.js reads a.hrs which is undefined on the mobile.js objects → NaN.

Fix: electric-meter.js patches window.calcKwhDay to accept both a.hrs and a.hours transparently. No existing files need editing. 

// Before (dashboard.js — breaks when mobile.js APPLIANCES wins):
function calcKwhDay(a) { return a.kw * a.hrs; }

// After (patched in electric-meter.js):
window.calcKwhDay = function(a) {
  var hrs = (a.hrs !== undefined) ? a.hrs : (a.hours || 0);
  return (a.kw || 0) * hrs;
};

🚀 Integration Steps

1
Copy the file to your JS folder
Place electric-meter.js in the same folder as mobile.js — typically /js/electric-meter.js or /public/js/electric-meter.js.
2
Add the script tag to index.html
Add after mobile.js — this is critical for the NaN patch to work correctly.
<!-- index.html — bottom of <body> -->
<script src="/js/config.js"></script>
<script src="/js/api.js"></script>
<script src="/js/auth.js"></script>
<script src="/js/views.js"></script>
<script src="/js/dashboard.js"></script>
<script src="/js/notifications.js"></script>
<script src="/js/mobile.js"></script>
<script src="/js/electric-meter.js"></script>  <!-- ← ADD THIS -->
3
Verify the Anthropic API proxy is working
The AI photo feature calls https://api.anthropic.com/v1/messages with an image payload. This is the same endpoint used elsewhere in the app. If your proxy strips the anthropic-version header, make sure it's still passing through — the module relies on your existing proxy setup. If the API call fails, the user sees a friendly warning and can enter kWh manually.
4
(Optional) Persist readings to your API instead of localStorage
By default readings are saved to localStorage under key emr_readings. To sync across devices, replace emrLoad() and emrSave() with API calls to your existing /api/meter-readings endpoint. The data shape is: 
// emr_readings structure:
{
  "storeId": [
    { "date": "2026-06-03", "kwh": 287.5, "source": "photo", "note": "" },
    { "date": "2026-06-02", "kwh": 273.0, "source": "manual", "note": "Meter reset" }
  ]
}

⚠ Photo storage: Meter photos are stored as base64 in localStorage. Large photos can exceed the ~5MB localStorage limit quickly. For production, consider uploading photos to your server and storing only the URL. The AI OCR runs before the photo is stored — if you want to discard the image after reading, call emrClearPhoto() after the AI fills the kWh field.
💡 KWH_DATA in mobile.js: Your existing KWH_DATA variable (historical meter readings Jan–May 2026) is separate from this module's real-time storage. You can seed emrLoad() with that historical data on first load to backfill the charts — or leave them as separate systems (historical vs. actual going forward).
